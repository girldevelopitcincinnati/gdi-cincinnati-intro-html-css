BUILDING BLOCKS

Anatomy of a website

Your Content
+ HTML (Structure)
+ CSS (Presentation)
= Your Website

A website is a way to present your content to the world, using HTML and CSS to display that content & make it look good.

Anatomy of a website

An Concrete Example

  • A paragraph is your content.
  • Wrapping your content in an HTML Element to mark it as a paragraph structure. 
    <p>A paragraph is your content.</p>
  • Making the font of your paragraph blue and 18pt is presentation:

    A block of text is your content.

Anatomy of an HTML Element

Element

  • An individual component of HTML
  • Example: Paragraph, heading, table, list, link, image, etc
<p>This is an HTML element, right here</p>

Anatomy of an HTML Element

Tag

  • Elements are made up of HTML tags.
  • All HTML tags have < and >
  • Marks the beginning & end of an element
  • Opening tag and Closing Tag
  • Tags contain characters that indicate the tag's purpose
<tagname>Stuff in the middle</tagname>
<p>This is a sample paragraph.</p>

Tag Breakdown

Anatomy of an HTML Element

Container Element

  • Can contain other elements or content
  • Has an opening and closing tag
<p>A paragraph is a container element.</p>

Empty Element

  • Cannot contain anything else
  • Does not have a closing tag
<br />
<img />
<hr />

Anatomy of an HTML Element

Attribute

  • Provides additional information about the HTML element
  • Class, ID, language, style, identity, source
  • Placed in the opening tag, before the right angle bracket

Value

  • Value is the value assigned to a given attribute
  • Values must be contained inside quotation marks

<span style="font-family: courier;">name="some value here"</span>

SECTION 3

GETTING STARTED

Folder Structure

All the files for your site should be stored within the same folder.

This includes:

  • HTML Files
  • CSS Files
  • Images
  • Script files
  • Anything else that will appear on your site

Note: File names should not include spaces or special characters. File names ARE case sensitive.

Doctype

The ﬁrst thing on an HTML page is the doctype, which tells the browser which version of the markup language the page is using.

Here's the doctype we use to tell the browser "Hey, this is HTML5, buddy." 
<!DOCTYPE html>

HTML Tag

After <doctype>, the page content must be contained between <html> tags. 

<!DOCTYPE html>
<html>

  All of our content is going to live between
  these two tags.

</html>

Head and Body Tags

The <head> contains meta (internal) information about the page.

The <body> contains the actual content of the page.

<!DOCTYPE html>
<html>

  <head>
    <title>Title of the page</title>
  </head>

  <body>
    The page content here.
  </body>

</html>
This is the basis of a complete HTML web page.

Nesting

Nesting is the placement of containing tags inside other containing tags. For example, you would put a set of <p> tags inside of the <body> tags. The <p> is now nested inside the <body>.

<body>
  <p>A paragraph inside the body tag</p>
</body>
Paragraph nested inside the <body> tag. 
<ul>
  <li>
    <p>A paragraph inside a list item</p>
  </li>
</ul>
Paragraph nested inside a list item.

Nesting

All elements "nest" inside one another. Whichever element opens first closes last.

Links

Links have three components

  • Tag: <a> </a>
  • href attribute: location of your link
  • alt attribute: description for screen readers
<a href="http://www.girldevelopit.com" alt="Girl Develop It Homepage">Girl Develop It!</a>

Girl Develop It!

Link tags can surround text or images.

Link Attributes

Links can also have attributes that tell the link to do different actions like open in a new tab, or launch your email program. 

<a href="home.html" target="_blank">Link Text</a>

Link opens in a new window or tab with target="_blank" 

<a href="mailto:sponsorships@girldevelopit.com">E-mail us!</a>

Adding mailto: directly before the email address will open the link in the default email program.

Relative & Absolute Paths

For links & images

Relative paths depend upon the page the link is on.

  • Links within the same directory need no path information. "filename.jpg"
  • Subdirectories are listed without preceding slashes. "images/filename.jpg"

Absolute paths refer to a specific file location, regardless of the current domain.

  • Typically used when pointing to a link that is not within your own domain.
  • "http://www.girldevelopit.com/chapters/cincinnati"

Tables

For more semantic oomph, you can wrap your header, body and footer rows in thead, tbody, tfoot tags.
